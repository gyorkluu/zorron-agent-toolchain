# Zorron 全局编码规范

## 通用原则

### 代码质量
- 代码应该清晰、简洁、可读优先
- 遵循所选语言的社区最佳实践和编码规范
- 函数/方法应保持单一职责，控制在合理长度内
- 变量命名应具有描述性，避免缩写（除非是广泛认可的缩写）

### Git 规范
- 提交信息使用 Conventional Commits 格式
- 提交前确保代码通过 lint 检查和测试
- 避免提交包含敏感信息的代码（密钥、密码、内部 URL）

### 文档规范
- 公共 API 和复杂逻辑必须添加注释
- README 应包含：项目描述、安装步骤、使用方法、配置说明
- 变更日志记录重要更新

### 安全规范
- 永远不要将 API 密钥、密码、证书等敏感信息硬编码在代码中
- 使用环境变量或密钥管理服务存储敏感信息
- 依赖项应定期更新以修复安全漏洞

## AI 协作准则

### 交互方式
- 在执行破坏性操作前，始终向用户确认
- 修改配置文件前，先备份原文件
- 提供清晰的解释说明每一步操作的目的

### 代码生成
- 生成的代码应与项目现有风格保持一致
- 包含必要的错误处理和边界检查
- 优先使用项目已有的依赖和工具

### 问题解决
- 先理解问题的根本原因，再提出解决方案
- 提供多种方案时，说明各自的优缺点
- 实施后验证解决方案的有效性
