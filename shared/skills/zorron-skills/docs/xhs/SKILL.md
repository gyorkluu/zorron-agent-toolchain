# 小红书（XHS）文案编写与引流规范

本指南旨在规范 AI 智能体在编写、润色和优化小红书（下文简称 XHS）笔记时的行为，确保文案天然适配小红书发布规范，并最大化防封避坑与导流效果。

---

## 一、 文本输出规范：绝对禁止 Markdown 格式

由于小红书 App 客户端不支持任何 Markdown 语法的渲染，所有生成或润色的笔记正文必须遵循以下**纯文本规范**，确保用户可以直接复制粘贴使用：

1.  **禁止代码块包裹**：不要使用 ```markdown ... ``` 或 ```text ... ``` 包裹笔记正文。
2.  **禁止粗体与斜体标记**：不要使用 `**` (双星号) 或 `*` (单星号) 进行文本强调。
3.  **禁止 Markdown 标题语法**：不要在正文中使用 `#`、`##`、`###` 等标题标记。
4.  **禁止 markdown 列表符号**：不要使用 `* ` 或 `+ ` 作为列表圆点，应改用 Emojis（如 1️⃣, 2️⃣, 📌, 🌟, 👉）或简单的连字符 `-`。
5.  **空行与分段**：使用双回车（两行换行符）来分隔段落，以模拟小红书上稀疏、易读的文字排版效果。

---

## 二、 语言风格规范：人性化中文重构

笔记正文必须遵循口语化、人格化原则，去除机械的 AI 写作痕迹（可配合 `humanizer-zh` 技能使用）：

1.  **采用真实人设**：使用第一人称“我”（比如：*“分享几个我自己天天在用的体验...”*），以热心博主或真实消费者的口吻进行对话。
2.  **句子长短交错（控制节奏）**：避免连续三句以上长度和结构相同的陈述句。多用短句和语气词（如：*“讲真”、“真的太香了”、“别慌！”*）。
3.  **删除高频 AI 词汇**：
    *   *禁止使用*：至关重要、深入探讨、格局、织锦、凸显、彰显、不可磨灭、璀璨、协同。
    *   *禁止使用夸张修饰*：顶级、赛博（除非特定数码圈暗号）、绝对、完美（改用更具体的词，如：很酷、踏实、省心）。
4.  **直接陈述事实**：不要做无意义的铺垫和多余的引导词，直接切入核心痛点或爽点。

---

## 三、 运营与防封避坑红线

在设计选题与引流策略时，必须主动融入以下防限流及合规导流策略：

### 1. 规避营销敏感词
*   在笔记文案中，**绝对不要**直接写出“微信”、“微信号”、“手机号”、“淘宝”、“链接”、“购买”、“下单”等引导外部交易的强敏感词。
*   网址链接（如域名 `mtb.gyork.fun`）不要在正文以文本格式大量打出，应改用图片或视频展示。

### 2. 三大合规导流打法
在笔记末尾的“行动指南”或“置顶评论”中，必须采用以下策略引导用户至你的私域/官网：
*   **打法 1（主页群聊引导）**：引导用户点击头像加入你的“小红书群聊”（群公告内放置引流微信或官网链接）。
*   **打法 2（小号评论置顶）**：主号发文，小号（如“助手账号”）在评论区发联系方式，主号置顶该评论。
*   **打法 3（图片/视频网址显影）**：在笔记配图、屏幕或视频的地址栏视觉元素中展示你的网址域名，文案只写*“浏览器输入xxx直达”*。

## 四、 封面视觉设计 UI 规范

如果需要为小红书设计或建议封面图，必须遵循以下 UI 原则：

1.  **设备场景化样机 (Contextual Mockups)**：不要用冷冰冰的网页截图，必须将系统 UI 套入 iPad/显示器中，并置于有灯光、咖啡等温馨极客书桌的实体环境内。
2.  **色调对比**：背景使用大面积暗色系（`#111116` / `#2a2a35`），亮部使用高饱和度的青色（Cyan `#00F0FF`）或绿色（Lime `#22C55E`）点缀，在小红书白底瀑布流中拉满对比度。
3.  **排版比例**：70% 图片，30% 文字。标题字应使用现代粗体无衬线字体（如得意黑、MiSans Bold），并加重阴影或透明磨砂底色板确保可读性。
